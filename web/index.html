학습 기준 2026년 3월까지

학교 전체 데이터와 건물별 과거 월 사용량, 2026년 3월 이전 일별 데이터를 학습에 사용합니다.

예측 구간 2026년 4월-6월

검증은 2026-04-15부터 2026-06-23까지 실제 건물 일별 전력과 비교합니다.

사용 가능한 입력 과거 사용량 + 예보 기상 + 건물 면적

LightGBM은 날짜, 최근 사용량, KMA 예보 기상값, 건물 면적/호실/층수 feature를 사용합니다.

제외한 입력 4월 이후 실제 총량 흐름

예측 대상 기간의 실제 학교 전체 일별 총량이나 일별 비중은 입력으로 쓰지 않습니다.

지표 읽는 법

오차 지표는 낮을수록 좋고, Bias는 0에 가까울수록 좋습니다.

2026년 4월 이후 예측은 매일 새로 들어오는 관측값과 그 시점까지 확인 가능한 예보만 쓴다고 가정합니다. 학습에는 실제 해당일 기상값을 쓰고, 검증 구간에는 전날까지 발표된 예보 기상값을 붙였습니다.

MAE

하루-건물 단위로 평균 몇 kWh 빗나가는지 보는 값입니다.

RMSE

큰 오차에 더 민감합니다. MAE보다 많이 크면 특정 날짜에서 크게 틀린 것입니다.

WAPE

전체 절대오차를 전체 실제 사용량으로 나눈 값입니다. 이번 검증의 주 지표로 사용합니다.

Bias

예측 방향의 치우침입니다. 양수는 과대예측, 음수는 과소예측입니다.

모델 작동 방식

이 페이지는 예측 모델의 입력, 계산 방식, 데이터 누수 방지 규칙을 함께 보여줍니다.

기준 모델 실시간 기준 월 프로파일 균등 일할

건물별 월 기준 사용량을 해당 월 날짜 수로 나눕니다. 가장 단순한 비교 기준입니다.

규칙 모델 요일+최근사용량 모델

월 프로파일 기준값에 전일 사용량, 최근 3일 평균, 최근 7일 평균, 전주 같은 요일 값을 섞습니다.

비교용 머신러닝 모델 LightGBM q50 walk-forward

과거 검증일 데이터를 순차적으로 학습해 다음 날짜를 예측합니다. random shuffle은 쓰지 않습니다.

현재 champion LightGBM + weather + area-v2

KMA 예보 기상 feature와 건물 면적 데이터를 결합한 후보가 이번 검증에서 가장 낮은 WAPE를 냈습니다. 학사일정 후보는 생성해 비교했지만 성능이 악화되면 champion에서 자동 제외합니다.

데이터 사용 시점 2026-04-10 예측에는 2026-04-09까지

당일 실제값과 이후 값은 사용하지 않습니다. 최근 사용량 feature는 항상 예측일보다 앞선 날짜에서만 계산합니다. LightGBM + weather + area 모델은 검증 구간에서 실제 기상이 아니라 전날까지 확인 가능한 예보 기상값을 사용합니다.

이번 실행 기록 LightGBM + weather + area: 예보 기상 + 건물 면적 결합

KMA APIHub 단기예보를 영남대학교 경산캠퍼스 기준 격자에서 일별 feature로 만들고, 건물 면적 엑셀의 전용면적, 공용면적, 호실수, 층수를 결합했습니다. 검증 구간은 17개 건물 1,190행이며, 학사일정 후보까지 함께 비교한 뒤 champion을 자동 선택했습니다.

입력 feature

  • 월 기준값: profile_monthly_kwh_mean, calendar_days_in_month, 월 프로파일 균등 일할
  • 날짜 정보: 요일, 주말 여부, 월, 일자
  • 최근 사용량: 전일 사용량, 최근 3일 평균, 최근 7일 평균, 전주 같은 요일
  • 핵심 고도화 입력: 냉방도일, 난방도일, 강수량, 강수 여부, 건물 면적, 호실수, 층수, 면적당 월 사용량
  • 기상: 냉방도일, 난방도일, 강수량, 강수 여부, 예보 발표 시점
  • 건물 면적: 건물 면적, 호실수, 층수, 면적당 월 사용량
  • 학사일정 후보: 수업 기간, 방학, 중간고사, 기말고사, 보강일, 강의평가 기간

fallback 규칙

  • 초반 날짜처럼 최근값이 부족하면 월 기준값으로 빈 값을 채웁니다.
  • 건물 면적을 매칭하지 못한 건물은 area_missing으로 표시하고 중위값 계열 fallback을 씁니다.
  • 학사일정은 과거 연도 전체 패턴을 무리하게 암기하지 않도록 굵은 이벤트 플래그만 씁니다.
  • 후보 모델은 같은 walk-forward 검증에서 WAPE와 MAE를 비교해 champion을 고릅니다.
