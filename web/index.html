학습 기준 2026년 3월 이전

2024~2025년 같은 월의 건물별 월 전력 프로파일을 사용합니다.

예측 구간 2026년 4월~6월

검증은 2026-04-01부터 2026-06-23까지의 실제 건물 일별 전력과 비교합니다.

사용한 입력 건물 월 프로파일 + 이전 사용량

기준선은 월 균등 일할을 쓰고, 새 모델은 건물별 이전 일별 사용량을 함께 사용합니다.

제외한 입력 4~6월 학교 일별 실측 흐름

예측 대상 기간의 실제 학교 전체 일별 총량과 일별 비중은 사용하지 않습니다.

지표 읽는 법

오차 지표는 낮을수록 좋고, Bias는 0에 가까울수록 좋습니다.

최근 사용량은 예측일 이전 데이터만 사용합니다. 2026년 4~6월 학교 전체 일별 실측 흐름 배분 방식은 데이터 누수 가능성이 있어 기준선과 새 모델 입력에서 제외했습니다.

MAE

하루-건물 단위로 평균 몇 kWh 빗나갔는지 보는 값입니다.

RMSE

큰 오차에 더 민감합니다. MAE보다 많이 크면 특정 날짜에서 크게 틀린 것입니다.

MAPE

사용량이 낮은 날짜에는 분모가 작아 크게 튑니다. 이 프로젝트에서는 MAPE는 주력 지표로 쓰지 않습니다.

WAPE

전체 절대오차를 전체 실제 사용량으로 나눈 값입니다. 공정성 baseline의 점 정확도는 이 값으로 봅니다.

Bias

예측 방향의 치우침입니다. 양수는 과대예측, 음수는 과소예측입니다.

모델 작동 방식

웹사이트만 보고도 두 예측 모델의 입력, 계산 방식, 데이터 누수 방지 규칙을 확인할 수 있습니다.

기준 모델 실시간 기준선: 월 프로파일 균등 일할

건물별 월 기준 사용량을 해당 월 날짜 수로 나눕니다. 예를 들어 월 기준값이 30,000 kWh이고 30일짜리 월이면 하루 예측값은 1,000 kWh입니다.

새 모델 요일+최근사용량 모델

월 프로파일 균등 일할 값을 기본값으로 두고, 같은 건물의 전일 사용량과 최근 며칠 흐름을 섞어 하루 예측값을 조정합니다.

비교용 머신러닝 모델 LightGBM q50 walk-forward

사람이 정한 고정 가중치 대신 과거 검증일 데이터를 학습해 다음 날짜를 예측합니다. 현재 변수만으로 기존 규칙 기반 baseline보다 지표가 얼마나 나아지는지 확인하기 위한 모델입니다.

데이터 사용 시점 2026-04-10 예측에는 2026-04-09까지

당일 실제값과 이후 값은 사용하지 않습니다. 최근 사용량 feature는 항상 예측일보다 앞선 날짜에서만 계산합니다. 학교 전체 일별 실측 흐름, 학교 전체 일별 총량, 학교 전체 일별 비중도 예측 입력에서 제외합니다.

입력 feature

  • 월 기준값: `profile_monthly_kwh_mean`, `calendar_days_in_month`, 월 프로파일 균등 일할
  • 날짜 정보: 요일, 주말 여부, 월, 일자
  • 최근 사용량: 전일 사용량, 최근 3일 평균, 최근 7일 평균, 전주 같은 요일
  • 과거 요일 패턴: 2026년 3월 이전 건물별 요일 패턴이 있으면 사용

fallback 규칙

  • 초반 날짜처럼 최근값이 부족하면 월 기준값으로 대신 채웁니다.
  • 과거 같은 월/요일 패턴이 부족하면 건물 월 평균으로 대신합니다.
  • 예측값은 음수가 되지 않도록 0 이상으로 제한합니다.
  • 이번 체크아웃에는 2026년 3월 이전 건물별 일별 행이 없어 요일 패턴은 월 기준값으로 fallback됩니다.

쉽게 말하면

기준선은 "이 조건이면 원래 이 정도는 쓸 건물"이라는 출발선입니다. 출발선이 너무 낮으면 그 건물은 절약하기 쉬워 보이고, 너무 높으면 절약해도 불리해집니다. 그래서 평균 오차보다 bias, coverage, 그룹 편향을 먼저 봅니다.
