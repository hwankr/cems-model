# 기준 모델 설계

## 목적

건물별 일일 전력 사용량 예측을 위한 첫 번째 전력 데이터 기반 기준 모델을 만든다.

이 기준 모델은 다음 질문에 답해야 한다.

- 과거 전력 사용량만으로 건물별 일일 사용량을 어느 정도 예측할 수 있는가?
- 어떤 건물에서 오차가 가장 큰가?
- 어떤 월이나 날짜 구간에서 기준 모델이 약한가?
- 이후 CEMS 프로젝트에서 사용할 수 있는 예측 파일은 무엇인가?

## 예측 대상

타깃 컬럼:

`usage_kwh_clean`

예측 단위:

`date`와 `building_name_recent` 조합당 1행

기본 검증 행:

`is_validation_target_clean == True`

## 기준 모델 방식

### 실시간 기준선: 균등 일할 기준 모델

건물의 과거 월별 사용량 프로필을 해당 월의 일수로 나누어 일일 사용량을 추정한다.

계산식:

```text
예측값 = profile_monthly_kwh_mean / calendar_days_in_month
```

이 모델은 일별 패턴을 고려하지 않는 단순 월평균 기준 모델이다.

### 제외한 방식: 학교 전체 일별 관측 패턴 배분

아래 방식은 2026년 4월 이후 실제 학교 전체 일별 전력 흐름을 사용하므로 실시간 예측 기준선에서 제외한다.

계산식:

```text
예측값 = profile_monthly_kwh_mean * school_total_day_weight_observed_month
```

이 방식은 사후 배분이나 복원 참고용으로만 볼 수 있다. 2026년 4월부터의 실제 운영 예측 상황에서는 사용할 수 없다.

## 평가 지표

다음 수준에서 성능 지표를 계산한다.

- 검증 행 단위
- 검증 기간 전체의 건물별 합계
- 건물과 월 조합
- 검증 기간 전체

필수 지표:

- MAE
- RMSE
- MAPE
- 평균 bias
- 전체 사용량 대비 bias 비율

MAPE는 실제값이 0이거나 결측인 행을 제외하고 계산한다.

- 예측 결과 파일이 저장된다.

- 오차가 큰 건물이나 기간을 명확히 확인할 수 있다.
