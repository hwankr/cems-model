# CEMS 학교 전력 모델 — 통합 설명서

## 0. 한 문단 요약

학교 전체의 **시간별 전력 검침값**에, **공공 기상(기상청)** 과 **학사일정** 을 더해 두 가지를 만들었습니다.

- **시스템 A — 과다사용 모니터**: "지금 이 시간, 평소보다 많이 쓰고 있나?"를 정상 범위(밴드)와 비교해 알려주고, 많이 쓸 때는 **트리 모델(SHAP)** 로 그 근거를 보여줍니다.
- **시스템 B — 절감 비교 데모**: "큰 설비 없이 이 정도 데이터만으로도, *자기 베이스라인 대비* 공정한 절감 비교가 된다"를 증명합니다.

두 시스템 모두 **서브미터·IoT 센서 같은 추가 설비가 전혀 필요 없습니다.** 대부분의 학교가 이미 가진 데이터만 씁니다.

## 2. 핵심 개념 쉽게 (비유 먼저)

- **정상 범위 = 일기예보 + 건강검진**
  - 일기예보가 "내일 낮 25~29도"처럼 *범위*로 말하듯, 모델도 "이 시간엔 보통 x~y kWh가 정상"이라고 범위로 답합니다.
  - 건강검진에서 정상수치를 벗어나면 빨간불이 켜지듯, 전기도 정상 상한을 넘으면 "과다사용"입니다.
- **베이스라인 = "원래라면 이만큼 썼을 것"**
  - 과거 패턴·날씨·학사일정으로 계산한 *기대 사용량*. 절감/과다사용을 재는 기준선입니다.
- **분위수 P10 / P50 / P90**
  - P50 = 중앙값(가장 그럴듯한 값). P10~P90 = 정상이라면 약 80%가 이 안에 들어오는 범위.
  - 정상 밴드 `[P10, P90]`, 실제가 P90을 넘으면 과다, P10보다 낮으면 (확실한) 절감.
- **왜 하나의 숫자가 아니라 범위인가?**
  - 전기 사용은 늘 조금씩 출렁입니다. 단일 예측값은 "맞다/틀리다"만 주지만, 범위는 "이 정도 출렁임은 정상"이라는 **불확실성**을 함께 알려줘서 오탐을 줄입니다.

## 3. 시스템 A — 과다사용 모니터

### 목적
"과거 패턴·날씨·학사일정으로 볼 때 이 시간엔 x~y kWh가 정상인데, 지금 z를 쓰는 중이고 상한보다 (z−y)만큼 더 쓴다. 그 이유는 …" 를 한눈에.

### 어떻게 만들었나
- **LightGBM 분위수 회귀**로 시간별 P10/P50/P90을 직접 예측.
- **day-ahead 계약**: 날짜·요일·시간 + 24시간 전·1주 전·최근 7일 같은시간 평균 + 예보 날씨 + 학사일정. **당일 직전 1시간 관측은 안 봄** → "지금 평소보다 많이 쓰는 중"을 탐지 가능(누수 방지).
- **CQR 보정**: 분위수 밴드가 좁아 적중률이 57%에 그쳤던 것을, 최근 28일(5월)로 컨포멀 보정해 **80% 적중하도록** 교정.
- **SHAP(트리 근거)**: 과다사용 시간마다 P50 기대값을 피처별로 분해(예: 요일 +521, 1주전 −395, 기온 …). 빨강=기대를 끌어올린 요인, 파랑=낮춘 요인.

### 결과 (검증 2026-06-01~06-20)
| 지표 | 값 |
| --- | ---: |
| 정상밴드 적중률(coverage) | **78.3%** (보정 전 59.0%, 목표 80%) |
| 과다사용 시간 | **72시간 (15%)** |
| 총 초과 사용량 | **17,287 kWh** |
| P50 예측 정확도(WAPE) | **7.57%** (naive 10.7~14.2% 능가) |

> June 적중률이 80%보다 살짝 낮은 것 자체가 "6월이 정상 범위보다 약간 높게 쓰였다"는 신호입니다.

### SHAP "근거"의 정확한 의미 (오해 주의)
SHAP는 *모델의 기대값(P50)이 왜 그 수준인지*를 설명합니다. 따라서 과다사용 시간에 대해 **(1) 맥락**(무엇이 기대 부하를 끌어올렸나: 폭염·시험기간 등) **+ (2) 초과분**(실제가 상한을 넘은, 모델이 예상 못한 추가 사용량)을 함께 봅니다. "폭염이라 원래 높게 기대됐는데 그보다도 더 썼다" vs "주말 야간이라 낮게 기대됐는데 크게 초과(강한 이상)"를 구분해 해석합니다.

### 웹 보는 법
`school_overuse_web/` (5탭): 개요 / 시계열 모니터(밴드+실제선+과다 빨강점) / 과다사용 분석(클릭 시 SHAP 막대) / 모델 정확도 / 용어 설명.

## 4. 시스템 B — 절감 비교 데모

### 목적 (두 가지를 증명)
1. **데이터 충분성**: 비싼 설비 없이 보유 데이터만으로 베이스라인이 충분히 정확하다.
2. **공정성**: 절대 kWh가 아니라 *자기 베이스라인 대비 %·면적당*으로 채점하면, 규모가 다른 학교도 공정하게 비교된다.

(학교 간 대결은 가정이고, 실제 한 학교 데이터로 방법론을 시연합니다.)

### 핵심 트릭 — "동결(frozen) 베이스라인"과 baseline chasing
일반 모델은 베이스라인에 **그 학교의 최근 사용량**을 입력으로 씁니다(시스템 A가 그렇습니다). 그런데 절약 대회에선 이게 **치명적**입니다:

> 학교가 꾸준히 절약하면 → 최근 사용량이 낮아짐 → **베이스라인도 따라 내려감(baseline chasing)** → "넌 원래 이만큼밖에 안 썼잖아"가 되어 절감 점수가 사라짐. 열심히 한 학교가 손해를 봅니다.

**해결 = M&V(IPMVP Option C) 스타일 동결 베이스라인**:
- 베이스라인을 **대회 전 데이터로만 학습**하고, 대회 기간 예측에 **그 기간 실제 사용량을 안 씀**.
- 피처 = 달력 + **동결 프로필**(`frozen_profile_kwh` = 기준기간의 요일×시간 평균, 한 번 계산해 고정) + 기상 + 학사일정. **사용량 lag·rolling을 일절 안 씀.**
- 동결 프로필이 부하 형상을 고정 앵커로 주므로, lag 없이도 정확도를 상당히 회복하면서 chasing을 차단합니다.

### 데이터 충분성 (정확도 증명)
| 지표 | 값 |
| --- | ---: |
| held-out WAPE (동결 P50) | **10.85%** |
| 단순 평균 프로필(naive) WAPE | 26.25% |
| → 모델이 naive 대비 | **약 59% 더 정확** |
| 정상밴드 적중률(coverage) | 80.4% |

"같은 요일·시간 평균"만 써도 26%인데, 거기에 날씨·학사일정을 더한 모델은 10.85%. **이 정도 데이터로 베이스라인이 이만큼 정확하다**는 증명입니다.

### 절감 채점 결과 (리포팅 2026-06-01~06-20)
| 지표 | 값 |
| --- | ---: |
| 순 절감량 | **−733.5 kWh** (약간 초과) |
| 절감률 (Σ절감 / Σ베이스라인) | **−0.05%** |
| 면적당 절감 | −0.0016 kWh/㎡ |
| 확실한 절감 시간 (실제<P10) | **100시간** |
| 초과 시간 (실제>P90) | 39시간 |

**주차 리더보드** (절감률 % 순위):
| 순위 | 라운드 | 절감률 | 면적당 | 순 절감 |
| :-: | --- | ---: | ---: | ---: |
| 1 | 2주차 (06-08~14) | **+1.96%** | +0.0229 | +10,264 kWh |
| 2 | 1주차 (06-01~07) | +0.41% | +0.0047 | +2,090 kWh |
| 3 | 3주차 (06-15~20) | −2.78% | −0.0292 | −13,088 kWh |

→ "여기선 같은 학교의 주차를 참가자로 줄세웠지만, **학교 A·B·C로 바꿔도 동일하게 동작**합니다."

## 5. 두 모델의 결정적 차이 (왜 다르게 만들었나)

| | 입력에 쓰는 과거 사용량 | 적합한 용도 | 정확도(WAPE) |
| --- | --- | --- | ---: |
| **operational** (운영형) | 직전 1시간까지 | 초단기 정확 예측 | 2.4% |
| **day-ahead** (시스템 A) | 24h·168h·7일평균 (당일 직전 1h 제외) | 과다사용 *탐지* | 7.57% |
| **frozen** (시스템 B) | 없음 (요일×시간 동결 프로필만) | 절감 *채점* (chasing 방지) | 10.85% (held-out) |

정확도는 operational > day-ahead > frozen 순이지만, **용도가 다릅니다.**
- 과다사용 탐지엔 day-ahead가 맞습니다(현재 급증을 안 봐야 잡아냄).
- 절감 채점엔 frozen이 맞습니다(절약해도 기준선이 안 따라와야 공정).
- lag를 뺄수록 정확도는 떨어지지만(7.57%→10.85%), 그 대가로 **공정성(chasing 차단)** 을 얻습니다.

## 6. 실제 학교 간 대결로 확장하려면

1. **학교마다 모델 1개**씩 동결 베이스라인 학습 (지금은 한 학교 시계열).
2. **점수 정규화**: 절감률(%) 또는 면적당 절감(kWh/㎡) — 규모 무관.
3. **베이스라인 동결 + 정기 재설정**: 대회 전 기간으로 고정하되, 학기·증축 등 구조 변화 시 재설정.
4. **측정 일관성**: 모든 학교가 같은 해상도·같은 누락 처리로 들어와야 공정.
5. **동점 처리**: 막상막하면 "절감이 베이스라인 오차범위(신뢰구간)를 넘었는가"로 판정.

## 7. 용어집 (핵심만)

- **분위수(Quantile)**: 크기순으로 줄 세웠을 때 특정 비율 위치의 값. P90 = 하위 90% 지점.
- **P10/P50/P90**: 정상이라면 P10 이상일 확률 90%, P50 근처가 가장 그럴듯, P90 이하 확률 90%. `[P10,P90]`에 정상 약 80%.
- **정상 밴드 / 과다사용**: `[P10,P90]`가 정상 범위. 실제>P90이면 과다, 초과량 = 실제−P90.
- **day-ahead(하루 전 기준)**: 당일 직전 1시간 관측을 안 쓰고 과거·날씨·학사일정으로 기대치를 만든 것.
- **동결(frozen) 베이스라인**: 대회 전 데이터로 고정한 기준선. 절감해도 따라 내려가지 않음.
- **baseline chasing**: 베이스라인이 줄어든 사용량을 따라 내려가 절감 점수가 사라지는 현상.
- **CQR (Conformalized Quantile Regression)**: 분위수 밴드의 폭을 held-out으로 보정해 적중률을 목표(80%)에 맞추는 기법.
- **M&V (Measurement & Verification, IPMVP)**: 절감 검증 표준. 베이스라인 회귀로 "절약 안 했다면 썼을 양"을 추정해 실제와 비교.
- **SHAP**: 트리 모델 예측을 피처별 기여도(kWh)로 분해.
- **WAPE**: 가중 절대 백분율 오차 = Σ|예측−실제|/Σ실제. 낮을수록 정확.
- **Pinball loss**: 분위수 예측 품질 지표. 낮을수록 좋음.
- **Coverage(적중률)**: 실제가 `[P10,P90]` 안에 든 비율. 잘 보정되면 ≈80%.
- **CDD/HDD**: 냉방·난방 도일. 기준온도 대비 더운/추운 정도의 누적 → 냉난방 수요 대리지표.
- **절감률 / 면적당**: 베이스라인 대비 절감 비율(%) / 절감을 연면적으로 나눈 값(kWh/㎡) — 규모 무관 공정 비교용.

## 8. 한계와 주의 (정직하게)

- **밴드는 "기대치" 기준**: 시스템 A 밴드는 day-ahead 기대치라, 직전 1시간까지 쓰는 operational 모델과 다릅니다. "지금 평소보다 많이 쓰는가" 용도에 맞춘 의도된 설계입니다.
- **동결 프로필의 형상 한계 (중요)**: 시스템 B의 동결 프로필은 *1년 평균* 요일×시간 형상이라, 6월 특유의 형상과 다릅니다. 그래서 **야간은 평균보다 낮게 써서 '절감'(100시간), 오후는 높게 써서 '초과'(39시간)** 가 상쇄되어 순 절감이 ≈0이 됩니다. → **시간 단위보다 일·주 단위 집계와 held-out 정확도(5월)** 가 더 대표성 있습니다. (개선안: 월별/계절별 동결 프로필.)
- **단일 학교 시연**: 진짜 학교 간 비교엔 학교별 시간 데이터가 추가로 필요합니다.
- **구조 변화 미반영**: 동결 베이스라인은 증축·용도변경 같은 구조 변화를 반영 못 하므로 정기 재설정이 필요합니다.
